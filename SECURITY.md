# Security Policy

## Supported Versions

Security fixes are prioritized for the latest main branch.

## Reporting a Vulnerability

Please do not disclose security issues publicly before a fix is available.

To report a vulnerability:

1. Send details to your private security contact channel for this project.
2. Include reproduction steps, impact, and suggested mitigations if available.
3. If possible, include affected endpoints, environments, and logs with secrets removed.

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Ongoing updates: at least weekly until resolution

## Disclosure Process

- We validate and triage the report.
- We prepare and test a fix.
- We coordinate release timing.
- We publish a security advisory after patch availability.
